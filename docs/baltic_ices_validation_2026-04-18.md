## Findings and Recommended Follow-ups

### Drift hits

- **cod (F ✗):** model F = 0.080, ICES F (tonnes-preferred) = 0.912 — model
  is **~91% below** ICES. The ICES column is now the SSB-weighted F over the
  tonnes-unit subset (`cod.27.22-24` alone, ~0.9 flat 2018-2021); the
  eastern index-unit stock `cod.27.24-32` is excluded and shown in the
  *excluded (unit mismatch)* column. **Framing** (post-review): this is a
  **scenario limitation, not a calibration choice** — OSMOSE-Baltic is not
  currently calibrated to reproduce the 2015-present cod collapse, and a
  model F matching ICES against a model biomass that fails to track the
  collapse would drive the population to extirpation in-model. Fix the
  biomass state before raising F. Follow-up: **document as an explicit
  scenario assumption** in `baltic_param-fishing.csv` header; do not claim
  this is a tuned calibration target.

- **cod (B ✗):** model envelope [60 000, 250 000] t does not overlap ICES
  [5 303, 12 896] t. ICES envelope is western Baltic cod only
  (`cod.27.22-24`, tonnes). **Review flag**: even accounting for
  total-biomass vs SSB (a 2-3× multiplier is typical for gadoids) and
  aggregation across both Baltic cod stocks, a plausible ICES-anchored
  total-biomass envelope for 2018-2022 lands near 30-100 kt, not 60-250 kt.
  The upper bound of 250 kt looks closer to the pre-2010 ecosystem state
  than to 2018-2022. **Follow-up** (separate calibration-tuning plan, out
  of scope here): narrow the upper end of the cod target, or re-label the
  target as "historical envelope, not 2018-2022 state." For now: keep cod
  in `B_KNOWN_EXCEPTIONS`; add a TODO in `biomass_targets.csv` pointing at
  this report section.

- **flounder (F ✗):** model F = 0.040, ICES F = 0.215 — model is ~81% below.
  `fle.27.2223` F in 2018-2022 fluctuated 0.09-0.33. Likely cause:
  coarse-grid calibration under-resolves flounder-concentrated lagoon
  habitats so the configured F compensates downward. This is a genuine
  grid-resolution trade-off and a defensible OSMOSE limitation. Follow-up:
  **document as scenario choice**; optional re-calibration once grid is
  refined.

### Non-circularity note (post-review)

This validator reads the 2024-advice snapshot. The model configuration
(`baltic_param-fishing.csv`, `biomass_targets.csv`) was authored from
earlier SAG pulls / literature values and does **not** consume this
snapshot as input. The comparison is therefore a one-way retrospective
check: 2024 advice (best current retrospective for 2018-2022 after
revisions) against an independently-derived model config. If a future
calibration loop ingests these snapshots as targets, that direction
would become circular — add a guard in the calibration script then.

## Known Limitations

- **Model aggregation.** `model_species_to_ices_stocks` aggregates four
  herring stocks into one `herring` species. Summed SSB/F is a
  simplification; per-stock Fmsy values cannot be combined linearly.
  Three of four herring stocks contribute to the envelope sum (all
  tonnes); the central Baltic stock `her.27.25-2932` is an index and is
  excluded.
- **Index vs tonnes.** Three stocks (`cod.27.24-32`, `her.27.25-2932`,
  `fle.27.2223`) report SSB as a relative biomass index rather than
  tonnes. The ICES SAG API mislabels these as `"tonnes"` in its per-row
  metadata; the snapshot manifest's `units_by_stock` field overrides
  this using Blim magnitude. Biomass envelope comparisons are silently
  skipped for these stocks; F-rate comparisons prefer the tonnes subset
  but fall back to the index subset when no tonnes stock is linked
  (flounder). The puller `scripts/_pull_ices_snapshots.py` raises if a
  stock cannot be classified — this is a hard gate, not a warning.
- **Mixed vintages.** `cod.27.22-24` is snapshotted from 2022 advice
  (last full assessment before the 2024 category-3 downgrade); all
  other stocks use 2024. Fine for SSB/F time series (2022 advice
  already covers 2018-2021 — only 2022 is lost), but **reference
  points (Blim/Bpa/Fmsy/MSYBtrigger) can move between benchmarks** —
  do not cross-compare RP values across vintages without flagging.
- **Coastal species.** Perch, pikeperch, smelt, stickleback have no
  ICES SAG assessment — validation impossible through this plan. A
  future `*-coastal-validation-plan.md` will cover DATRAS-CPUE /
  national-survey alternatives.
- **Full-coverage envelope.** `_ices_ssb_envelope` uses the
  intersection of years for which **all** linked tonnes-unit stocks
  report SSB. A stock assessed only from 2020 will drop 2018-2019 from
  the envelope for its species rather than undercounting them. If the
  resulting window shrinks below three years the result is likely
  noise — report as "insufficient overlap" rather than drift.
- **Weighted F over tonnes subset.** `_ssb_weighted_f` is computed over
  a single-unit subset of stocks (tonnes preferred, index fallback) to
  avoid the silent tonnes-dominance problem. `FComparison.excluded_
  index_stocks` surfaces which stocks were dropped. For species with
  mixed units (cod, herring), per-stock F is NOT currently reported in
  a separate table — add one in a follow-up if reviewers want
  per-stock granularity.
